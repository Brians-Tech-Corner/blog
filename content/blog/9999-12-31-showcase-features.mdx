## Mermaid Diagrams

Create flowcharts, sequence diagrams, and more using Mermaid syntax.

### Flowchart

<Mermaid>
{`graph TD
    A[Start] --> B{Is it working?}

    B -->|No| D[Debug]
    D --> E[Fix Issue]
    E --> B
    C --> F[Deploy]`}
</Mermaid>

## Series Posts

To create a series, add these fields to frontmatter:

```yaml
series: "my-series-name"
seriesOrder: 1
```

This post would then show navigation to other posts in the same series.

## Draft Posts

Set `draft: true` in frontmatter to hide posts from production (they'll only show in development).

## Tags

Add tags to frontmatter for categorization:

```yaml
tags: ["kubernetes", "docker", "devops"]
```

## Tips for Writing

1. **Use descriptive titles** - They show up in search results and social shares
2. **Add descriptions** - Helps with SEO and provides context
3. **Use code blocks liberally** - They're syntax-highlighted and copyable
4. **Add diagrams for complex concepts** - Mermaid diagrams help visualize architecture, flows, and processes
5. **Break up text** - Use headings, lists, and callouts to improve readability
6. **Test in development** - Use `draft: true` while writing and testing

### Using Mermaid Diagrams

Wrap Mermaid syntax in a `<Mermaid>` component with template literals:

```jsx
<Mermaid>
{`graph TD
    A[Start] --> B[End]`}
</Mermaid>
```

Supported diagram types:
- Flowcharts (`graph TD`, `graph LR`)
- Sequence diagrams (`sequenceDiagram`)
- State diagrams (`stateDiagram-v2`)
- Class diagrams (`classDiagram`)
- Entity-relationship diagrams (`erDiagram`)
- And more! See [Mermaid docs](https://mermaid.js.org/) for full syntax
